Chapter 6: Programming Languages
Terms Definitions
Machine Language Machine language is a collection of binary digits or bits that the computer reads and interprets. Machine language is the only language a computer can understand.
Debugging Locating and correcting errors in machine languages within a program.
Assembler Converts mnemonic expressions into machine language instructions.
Assembly Language A mnemonic system for representing programs, known also as second-generation languages.
Translator Translates programs expressed in these high-level primitives into machine-language programs.
Compiler Translation programs that compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter An alternative to translators, emerged as another means of implementing third-generation languages.
Cross-platform Software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.
Natural Language Languages such as English, German, and Latin.
Formal Language Languages such as programming languages, like Java, C, and Python.
Machine Independence A term applied to software that is not dependent on the properties of a particular machine, and can therefore be used on any machine.
High-Level Programming Language Provides more abstractions for the programmer and make it easier for people to read and write a program.
Object-Oriented Programming (OOP) Following this paradigm, a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects.
Scripted Language A subset of the imperative programming languages is the collection of languages known as scripting languages. These languages are typically used to perform administrative tasks rather than to develop complex programs.
Data Types (Integer, Float, String)
  1. Data Types: Declarative statements that require the programmer to describe the type of data that will be stored at the memory location associated with the variable.
  2. Integer: Refers to numeric data consisting of whole numbers.
  3. Float: Refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation.
  4. String: A data type used in programming, that is used to represent text rather than numbers.
Primitive Data Type The data types that are included as primitives in a programming language, such as int for integer and char for character.
Data Structure The conceptual shape or arrangement of data. For example, text is normally viewed as a long string of characters whereas sales records might be envisioned as a rectangular table of numeric values, where each row represents the sales made by a particular employee and each column represents the sales made on a particular day.
Python Collection Data Structures (List, Tuple, Set, Dictionary)
  1. List: Lists are used to store multiple items in a single variable. Created using square brackets.
  2. Tuple: Used to store multiple items in a single variable, ordered and unchangeable. Written with round brackets.
  3. Set: Used to store multiple items in a single variable. It is unordered, unchangeable, and unindexed. Written with curly brackets.
  4. Dictionary: Used to store data values in key:value pairs. It is ordered, changeable and do not allow duplicates. Written with curly brackets, and have keys and values.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable A variable declared within a function, meaning that it can be referenced only within that function.
Global Variable Variables whose scopes are not restricted to a particular part of a program.
Source Program The program in its original form.
Object Program The program in its translated version.
