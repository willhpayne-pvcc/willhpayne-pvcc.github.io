Software Life Cycle: Development phase Involves 4 steps including requirements analysis, design, implementation, and testing
  • Requirements analysis:To specify what services the proposed system will provide, to identify any conditions ( time constraints, security, etc.) on those services, and to define how the outside world will interact with the system.
  • Design:Creating a plan for te construction of the proposed system. Developing a solution to the problem.
  • Implementation: the actual writing of programs, creation of data files, and develpment of databases.
  • Testing:traditionally determining final software is compatible wih the software requirements specification.
modular implementation The devision of software into manageable units called modules.

Software testing: Verifying softwware
  • glass box testing: The tester is aware of the interior structure of the software and uses this knowledge when designing the test.
  • black box testing: Tests that do not rely on knowledge of the software's interior composition.
  • beta testing:A black box test, shere a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
  • alpha testing: Similar testin like beta testing but is done at the develper's site.
Documentation: To learn to use and a=maintain it.
  • user documentation: To explain the features of the software and describe how to use them.
  • system documentation : To describe the software's internal composition so that the software can be mainatianed later in its life cycle.
  • technical documentation: To describe how a software system should be installed and serviced.

  • software disclaimer: State the limitations of their liability.
  • best defense against software liability claims ( see last sentence in the chapter): To apply sound software engineering principles during the software's development, to use a level of care compativle with the software's application, and to produce and maintian records that validate these endeavors.
